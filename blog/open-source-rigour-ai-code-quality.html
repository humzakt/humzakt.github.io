I Built an Open-Source AI Code Quality Tool — 38 Releases in 5 Weeks

Feb 8, 2026 · 6 min read · Humza Tareen
Open Source AI Code Quality TypeScript Developer Tools

The Problem

AI coding assistants are powerful but messy. They skip tests, introduce security vulnerabilities, create retry loops, and ignore project conventions. I wanted a tool that catches these issues before they make it to production.

What Rigour Does

  • Universal AST-based code analysis (TypeScript, Python, Go, Java, and more)
  • Security vulnerability detection (prototype pollution, SQLi, XSS, CSRF, hardcoded secrets)
  • Staleness detection (catches when AI uses deprecated patterns)
  • Retry loop detection (prevents AI from trying the same broken fix)
  • Supervisor Mode for iterative quality improvement
  • IDE-agnostic: Cursor, Claude Code, Gemini, Codex/Aider, Windsurf

The Release Velocity

v1.0.0 → v2.18.1 in 5 weeks. 38 releases (25 features, 12 bug fixes). Published to MCP Registry. Semantic versioning with automated releases via GitHub Actions.

Architecture Decisions

  1. Config-driven gates — rigour.yml defines what quality checks run. Teams customize without forking.
  2. MCP protocol support — works as a Model Context Protocol server, so any MCP-compatible IDE can use it.
  3. Fix packets — when Rigour finds an issue, it generates a structured "fix packet" the AI can consume and apply.

The Hardest Part

It wasn't the code analysis — it was making the tool work across different AI agents. Each IDE/agent has its own way of handling instructions. Building a universal interface that works with all of them was a design challenge I didn't anticipate.

The project is open source at github.com/rigour-labs/rigour.
