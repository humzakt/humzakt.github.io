Building an RL Training Arena for AI Code Agents on GCP

Feb 8, 2026 · 7 min read · Humza Tareen
Reinforcement Learning AI GKE Cloud Build Python

The Problem

We needed AI agents that could learn from their mistakes when solving coding tasks. Not just prompt engineering — actual reinforcement learning with structured rewards.

What I Built

Executor Service (81% test coverage)

  • Gym-style RL API: create episodes, execute steps, compute rewards
  • Cloud Build integration for sandboxed code execution
  • Redis-based episode state management
  • Firestore trajectory logging for training data
  • Custom exception hierarchy for proper error propagation

Preprocessor Service

  • 15-step preprocessing workflow for coding tasks
  • Dynamic verifier and reward configuration generation
  • Repository analysis and baseline test execution

Architecture Decisions

  1. Hexagonal architecture — Domain logic is completely isolated from infrastructure. Switching from Cloud Build to local Docker? Change the adapter, not the domain.
  2. Redis for episode state — Episodes are short-lived but need fast read/write.
  3. Firestore for trajectories — Training data needs to persist long-term and be queryable.
  4. Cloud Build sandboxing — Each agent execution happens in an isolated Cloud Build step. No cross-contamination between evaluations.

The Reward System

reward = (tests_passed / total_tests) * base_reward
        + style_bonus
        - time_penalty
        - error_penalty

Getting the reward shaping right took 3 iterations. Too sparse → agents don't learn. Too dense → agents game the metric.

Scale: 55 GCS buckets store task artifacts, results, and evaluation checkpoints. 3 GKE clusters with auto-scaling node pools handle the compute.
