Building an Auth Gateway with Admin Dashboard: JWT, OIDC, and Notifications Admin

Feb 8, 2026·7 min read·Humza Tareen
AuthenticationAPI GatewayJWTKongReactCloud Run

What This Service Does

The Guard Service is the front door to the platform. Handles auth (Google OIDC, JWT, API keys), authorization, API routing via Kong, admin dashboard for notifications/subscriptions/delivery logs. 10 PRs and 22 security issues filed.

Notifications Admin Dashboard (PRs #44-#48)

5 PRs in a single day: PR #44 fixed crashes with missing notification IDs. PR #45 row-click details modal + pnpm lockfile (+4,053 lines). PR #46 subscription "All vs Active" filter + sortable tables. PR #47 table pagination + delivery log filters. PR #48 admin guide documentation.

Route Cache Invalidation Bug (PR #43)

API gateway caches routes for performance. Admin updates didn't invalidate cache. Stale routes served for 5 min. Deleting a service returned 200 instead of 404. Fix: explicit cache invalidation on mutations + proper 404.

Invite Auth Flow (PR #52)

Two bugs: invite URL pointed to wrong OAuth callback, accepting invite could downgrade existing user's role. Fix: correct URL construction + role comparison (only upgrade, never downgrade).

Security Audit (22 Issues)

Critical: hardcoded Cloud Run URL in OpenAPI spec, missing security scanning. High: JWT in localStorage (XSS), missing POSTGRES_HOST in prod, no error boundaries, console.log leaking data. Medium: missing OAuth state validation, no DB enum validation, missing indexes.

Pre-Commit Hooks (PR #49)

+15,403/-11,781 lines. Added formatting + compile/typecheck hooks for backend (Python/ruff) and frontend (Prettier/ESLint).
