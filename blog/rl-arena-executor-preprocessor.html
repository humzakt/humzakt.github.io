Building an RL Training Arena: Gym-Style API, Cloud Build Sandboxing, and 81% Test Coverage

Feb 8, 2026·8 min read·Humza Tareen
Reinforcement LearningGKECloud BuildRedisPython

What it does

This is a Gym-style RL environment for training AI coding agents. I built it from scratch across 4 PRs, totaling 12,000+ lines of production code.

Executor Service (PRs #31, #32)

2,800+ lines across 53 files with 81% test coverage.

Gym-Style API

  • POST /episodes — Create a new training episode
  • POST /episodes/{id}/steps — Execute an action step
  • GET /episodes/{id}/reward — Retrieve reward signal

Cloud Build Sandboxing

Isolated execution environment with:

  • Isolated containers with no network access
  • Hard timeout enforcement
  • Resource limits (CPU, memory)
  • Read-only filesystem except for scratch space

State Management

  • Redis: Episode state (fast, ephemeral)
  • Firestore: Trajectories (persistent, queryable)
  • Cloud SQL: Task configuration

Custom Exception Hierarchy

  • EpisodeNotFoundError
  • ExecutionTimeoutError
  • SandboxViolationError
  • And more for comprehensive error handling

Preprocessor Service (PR #28)

3,900+ lines across 41 files. "The Task Chef" — transforms raw repositories into training-ready tasks.

15-Step Pipeline

  1. Repo cloning: Fetch source code from Git
  2. Language detection: Identify primary programming language
  3. Dependency resolution: Parse package files, build dependency graph
  4. Test discovery: Find all test files and test cases
  5. Baseline execution: Run tests to establish ground truth
  6. Verifier generation: Create test verification logic
  7. Reward config: Define reward function parameters
  8. Task decomposition: Break complex tasks into subtasks
  9. Difficulty estimation: Score task complexity
  10. Resource calculation: Estimate CPU/memory/time requirements
  11. Sandbox config: Generate Cloud Build configuration
  12. Prompt generation: Create agent instruction prompts
  13. Validation rules: Define success/failure criteria
  14. Artifact packaging: Bundle all assets
  15. GCS upload: Store in Google Cloud Storage

Architecture

Hexagonal architecture for infrastructure portability. The core domain logic is decoupled from GCP-specific implementations, making it easier to test and potentially migrate to other cloud providers.
